ExParsec.Base

  • Summary
  • Functions

Provides fundamental combinators and parsers.

Source

Summary
↑

attempt(parser)

Identical to applying parser normally, except that if applying parser results in a fatal error, it will be turned into a regular error
between(parser1, parser2, parser3)

Applies parser1, parser2, and parser3 in sequence. Returns the result of parser2
bind(parser, function)

Applies parser and passes its result as the only argument to function. function is expected to return a parser. That parser is then applied and its result is returned
both(parser1, parser2, function)

Applies parser1 and parser2 in sequence. Passes the result values as two arguments to function. function‘s return value is returned as the result
choice(parsers)

Tries to apply each parser in parsers until one succeeds. This is a variant of either/2 generalized for any number of parsers
describe(parser, name)

Applies parser. If it fails, its errors are propagated in addition to an extra error generated based on name of the form "expected #{name}"
either(parser1, parser2)

First tries to apply parser1. If that fails, tries to apply parser2. If that fails, this combinator fails. Otherwise, returns the first successful result value obtained
eof()

Only succeeds at the end of the input data
fail(message)

Fails with the given error message
fail_fatal(message)

Fails fatally with the given error message
get_position()

Returns the current position as result
get_state()

Returns the user state as result
ignore(parser)

Applies parser and discards its result
label(parser, name)

Applies parser. If it fails, replaces its error with one generated based on name of the form expected #{name}
many(parser)

Applies parser as many times as possible. Returns all results in a list

Applies parser and passes its result to function. function‘s return value is returned as the result
option(parser)

Applies parser if possible. Returns a tuple containing :ok and the result, or nil if parser could not be applied

Applies parser1 and parser2 in sequence. Returns the result of both parsers as a tuple
pair_left(parser1, parser2)

Applies parser1 and parser2 in sequence. Returns the result of parser1
pair_right(parser1, parser2)

Applies parser1 and parser2 in sequence. Returns the result of parser2
pipe(parsers, function)

Applies each parser in parsers. Passes all result values in a list to function. function‘s return value is returned as the result
return(value)

Returns value as result

set_state(state)

skip(parser)

Applies parser if possible. Discards the result
skip_many(parser)

Applies parser as many times as possible. Discards the results
times(parser, n)

Applies parser to the input data n times. Returns results in a list
update_state(updater)

Updates the user state by applying updater to it
