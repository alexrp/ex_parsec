ExParsec.Input protocol

  • Summary
  • Functions
  • Types

Represents a source of input data for parser functions.

This protocol should be implemented for types that can be used to feed data (codepoints, tokens, etc) to parser functions. Note that an implementation must only emit one kind of input data, not several.

This is a relatively low-level interface that doesnâ€™t even provide position tracking. In general, beyond implementing this protocol, you should not be using it.

get(input, opts \\ [])
(function) # ↑

Fetches data from the input. If no more data is available, :eof is returned. If invalid input data is encountered, a tuple containing :error and a reason is returned. Otherwise, returns a tuple containing the advanced input and the fetched data.

opts can be used for implementation-specific options.
