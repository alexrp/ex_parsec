ExParsec v0.2.1 → Overview → ExParsec → Text

ExParsec.Text

  • Summary
  • Functions

Provides common parsers that operate on text.

These parsers all require that the input data is codepoints.

↑

alphanumeric()

Expects and parses any alphanumeric character (i.e. ?A .. ?Z, ?a .. ?z, and ?0 .. ?9)
any_char()

Parses a codepoint. Returns the codepoint as result
bin_digit()

Expects and parses any digit in ?0 .. ?1
char(codepoint)

Expects and parses the given codepoint. On success, returns the codepoint as result
digit()

Expects and parses any digit in ?0 .. ?9
hex_digit()

Expects and parses any digit in ?0 .. ?9, ?A .. ?F, and ?a .. ?f
letter()

Expects and parses any letter in ?A .. ?Z and ?a .. ?z
lower()

Expects and parses any letter in ?a .. ?z
newline()

Expects and parses a newline sequence. This can either be a `” “` or a `” “followed by“ “. Either way, returns“ “` as result
none_of(codepoints)

The opposite of one_of/1: Expects a codepoint that’s not in codepoints. Otherwise, works like one_of/1
oct_digit()

Expects and parses any digit in ?0 .. ?7
one_of(codepoints)

Expects and parses a codepoint that’s present in codepoints, which can either be a list of codepoints, or a string that’s converted to a list of codepoints
satisfy(name, function)

Expects and parses a codepoint that satisfies the criteria required by function. name is used for error message generation
space()

Expects and parses any white space character
spaces()

Parses as many white space characters as possible
string(string)

Expects and parses the given string. On success, returns the string as result
tab()

Expects and parses a tab (" ") character
upper()

Expects and parses any letter in ?A .. ?Z
